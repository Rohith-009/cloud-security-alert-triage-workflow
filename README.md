# Cloud Security Alert Triage Workflow

## Overview

This project presents a UX/UI design solution for a Cloud Security Alert Triage Workflow. The design helps security engineers efficiently prioritize, investigate, and resolve cloud security alerts.

## Problem Statement

Cloud security teams receive hundreds of alerts daily. Many alerts lack sufficient context, resulting in alert fatigue and slower response times. This project aims to streamline the alert investigation process through a centralized workflow.

## User Persona

### Cloud Security Engineer

Responsibilities:

* Monitor cloud security alerts
* Investigate incidents
* Assess risks
* Coordinate remediation

Goals:

* Reduce investigation time
* Prioritize critical alerts
* Resolve incidents efficiently

## Wireframes

### 1. Alert Dashboard

Displays alerts across cloud environments with filtering and search capabilities.

### 2. Investigation View

Provides alert context, affected resources, activity timelines, and response actions.

### 3. Resolution Center

Offers remediation recommendations, root cause tracking, and alert closure workflow.

## Feature Prioritization

### P0 - Must Have

### P1 - Should Have

### P2 - Nice to Have

## Success Metrics

* Mean Time to Investigate (MTTI)
* Mean Time to Resolve (MTTR)
* Alerts Resolved per Engineer
* Reduction in Unresolved Critical Alerts
* User Satisfaction Score
